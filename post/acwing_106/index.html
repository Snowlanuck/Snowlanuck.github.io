AcWing_106动态中位数

· 2021-01-07 · # 算法

题目地址：106. 动态中位数 - AcWing题库

题目描述

依次读入一个整数序列，每当已经读入的整数个数为奇数时，输出已读入的整数构成的序列的中位数。

输入格式

第一行输入一个整数PPP，代表后面数据集的个数，接下来若干行输入各个数据集。
每个数据集的第一行首先输入一个代表数据集的编号的整数。
然后输入一个整数MMM，代表数据集中包含数据的个数，MMM一定为奇数，数据之间用空格隔开。
数据集的剩余行由数据集的数据构成，每行包含10个数据，最后一行数据量可能少于10个，数据之间用空格隔开。

输出格式

对于每个数据集，第一行输出两个整数，分别代表数据集的编号以及输出中位数的个数（应为数据个数加一的二分之一），数据之间用空格隔开。
数据集的剩余行由输出的中位数构成，每行包含10个数据，最后一行数据量可能少于10个，数据之间用空格隔开。
输出中不应该存在空行。

数据范围

$ 1 ≤ P ≤ 1000 $,

$ 1 ≤ M ≤ 9999 $

输入样例：

3 
1 9 
1 2 3 4 5 6 7 8 9 
2 9 
9 8 7 6 5 4 3 2 1 
3 23 
23 41 13 22 -3 24 -31 -11 -8 -7 
3 5 103 211 -311 -45 -67 -73 -81 -99 
-33 24 56

输出样例：

1 5
1 2 3 4 5
2 5
9 8 7 6 5
3 12
23 23 22 22 13 3 5 5 3 -3 
-7 -3

题解

题目的大概意思是说，每读入一个数且当前读入的次数是奇数时，求整个数组的中位数是多少。

看个题目的样例：

1 2 3 4 5 6 7 8 9

第一次读入1 时，中位数就是1 （数组：{ 1 }）

第二次读入2时，不管

第三次读入3时，中位数就是2 （数组：{1，2，3}）

第四次读入4时，不管

第五次读入5时，中位数就是3（数组：{1，2，3，4，5}）

...

从题目给出的数据范围和时限来看，我们需要用差不多O(1)的时间实现才行。

那么有什么方法可以实现快速求中位数这个操作呢？

答案是对顶堆。

对顶堆由一个大根堆和一个小根堆实现，维护另大根堆永远比小根堆多一个元素，那么大根堆的堆顶就是中位数，具体原理和代码见下。

代码：

#include <bits/stdc++.h>
using namespace std;

int main()
{
    int T, n, m; cin >> T;
    while (T -- && cin >> m >> n)
    {
        printf("%d %d\n", m, (n + 1) / 2);
        priority_queue<int> down;
        priority_queue<int, vector<int>, greater<int>> up;
        
        int cnt = 0;
        for (int i = 1, x; i <= n; i ++)
        {
            scanf("%d", &x);
            if (down.empty() || x <= down.top()) down.push(x);
            else up.push(x);
            
            if (down.size() > up.size() + 1) up.push(down.top()), down.pop();
            if (up.size() > down.size()) down.push(up.top()), up.pop();
            
            if (i % 2) 
            {
                printf("%d ", down.top());
                if (++ cnt % 10 == 0) puts("");
            }
        }
        
        if (cnt % 10) puts("");
    }
    
    return 0;
}
